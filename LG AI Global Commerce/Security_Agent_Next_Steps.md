# Security Guardian: 상용화 및 고도화를 위한 Next Steps

현재 프론트엔드-AI 간의 개념 증명(PoC) 및 데모 시뮬레이션으로 완성된 Security Guardian을, 실제 라이브 서버에서 24시간 스스로 업무를 수행하는 **완전 자율형 AI 에이전트(Autonomous Agent)**로 발전시키기 위한 5단계 로드맵입니다. 해커톤 발표 시 프로젝트의 확장성과 실현 가능성을 어필하는 핵심 자료로 활용할 수 있습니다.

---

## 1. 실제 트래픽 로그 수집기 (Interceptor) 연동
- **현재:** 버튼을 클릭하여 가상의 위협 데이터를 발생시킴.
- **실제 구현:** 백엔드 API 서버(Spring Boot / Node.js) 앞단에 `Security Filter (Middleware)` 코드를 추가합니다. 모든 접속자의 IP 주소, 접속 빈도, 입력 페이로드(Payload)를 Redis 등 인메모리 DB에 실시간으로 수집 및 스트리밍합니다.

## 2. 임계치 기반의 AI 자동 기상 시스템 (Auto-Trigger)
- **현재:** 어드민 대시보드에서 사람이 명령어를 입력해야 시뮬레이션 시작.
- **실제 구현:** "특정 IP에서 1초에 50번 이상 접속" 하거나 "입력창에 의심스러운 특수문자(`'` 또는 `;`)가 포함"되는 등 룰 엔진의 임계치(Threshold)를 초과하는 즉시, 시스템이 자동으로 Gemini AI(보안 요원)를 호출하여 해당 로그의 분석을 의뢰합니다.

## 3. 실제 방어 기능 수행 (Function Calling 구현)
- **현재:** AI가 화면에 "차단했습니다"라고 텍스트만 출력함.
- **실제 구현:** AI에게 백엔드 코드를 실행할 수 있는 권한(Tool Calling)을 부여합니다. AI가 해킹으로 판별하고 `block_ip(ip="123.45.xx.xx")`라는 함수를 호출하면, 서버의 방화벽 블랙리스트 DB에 해당 IP를 즉시 추가하여 접속을 원천 차단합니다.

## 4. 휴먼 인 더 루프 (Human-in-the-Loop) & 알림 시스템
- **목적:** AI의 오탐(False Positive)으로 인해 정상적인 대규모 트래픽(예: 이벤트 접속자)을 차단하는 것을 방지.
- **실제 구현:** 단순 매크로 공격은 AI가 알아서 방어하되, 대규모 국가 대역 차단 등 치명적인 조치를 취하기 직전에는 Slack이나 어드민 화면으로 알림을 보냅니다. *"관리자님, 현재 심각한 공격을 탐지했습니다. 해당 대역을 차단할까요? [예/아니오]"* 와 같이 사람의 최종 승인을 거치도록 설계합니다.

## 5. 일일 보안 브리핑 자동화 (CRON Job)
- **목적:** 관리자의 개입 없이 보안 상태를 정기적으로 리포팅.
- **실제 구현:** 매일 아침 9시, AI가 간밤에 발생한 서버 접속 로그와 방어 내역을 스스로 분석합니다. 우리가 구현했던 '보안 보고서 자동 생성 기능'을 스케줄러에 연동하여 경영진에게 요약된 보안 리포트를 이메일로 자동 발송합니다.
